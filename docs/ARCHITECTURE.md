# Architecture & Design — Milestone 1

## 1. Objective (from project spec)
Build an AI-powered Sports Injury Risk Detection Platform that analyzes athlete
movement videos to identify biomechanical issues, detect abnormal movement patterns,
assess injury risk, and predict potential injuries before they occur.

Milestone 1 lays the foundation: authentication, RBAC, athlete profile management,
and the base frontend/backend setup. Video processing, pose estimation, and ML models
are built in later milestones (see project spec, Milestones 2-4).

## 2. High-level system architecture (current + planned)

```
[React Frontend] --HTTP/JWT--> [FastAPI Backend] --SQL--> [PostgreSQL / SQLite]
                                      |
                          (Milestone 2+: Pose Estimation,
                           Biomechanics, Risk Prediction services)
```

This repo currently implements the left two boxes. The full target architecture
(microservices for video processing, pose estimation, biomechanics analysis, risk
scoring, etc.) is described in the original project brief and will be added
incrementally in later milestones.

## 4. Authentication & RBAC

- Passwords are hashed with bcrypt (via passlib).
- Login issues a JWT (`/auth/login`), containing the user id and role.
- Protected routes use `get_current_user` to decode the token.
- Role-restricted routes use `require_roles([...])`, matching the project's
  Roles list: Athlete, Coach, Physiotherapist, Sports Scientist, Administrator.

Example: only coach/physiotherapist/sports_scientist/administrator roles can list
all athletes (`GET /athletes`); an athlete can only read/update their own profile
(`GET/PUT /athletes/me`).

## 5. API endpoints (Milestone 1)

| Method | Path | Access | Purpose |
|---|---|---|---|
| POST | /auth/register | public | create a user account |
| POST | /auth/login | public | get a JWT access token |
| GET | /auth/me | authenticated | get current user |
| GET | /athletes/me | athlete | view own profile |
| PUT | /athletes/me | athlete | update own profile |
| GET | /athletes | coach/physio/scientist/admin | list all athlete profiles |
| GET | /athletes/{id} | coach/physio/scientist/admin | view one athlete profile |

Interactive API docs are auto-generated by FastAPI at `/docs` once the server is running.

## 6. Tech stack (matches project spec section 7)

- Backend: Python, FastAPI
- Frontend: JavaScript, React (Vite), React Router, Axios
- Database: SQLite for local dev (drop-in swap to PostgreSQL for production —
  just change `DATABASE_URL` in `.env`)
- Auth: JWT (python-jose) + bcrypt password hashing (passlib)

## 7. What's next (Milestone 2 preview)

- Video upload & processing pipeline (OpenCV, FFmpeg)
- Biomechanical analysis (joint angles, symmetry, posture)

These require the datasets listed in the project spec (Human3.6M, MPII, COCO
Keypoints, SportsPose, FIFA Injury dataset for reference).
